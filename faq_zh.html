⛓️ 為什麼儲存在區塊鏈中的資料「絕對無法被刪除或修改」？

要理解區塊鏈資料的不可篡改與永久性，最完美的範例就是 Bitcoin (比特幣) 區塊鏈。自從 2009 年比特幣的「創世區塊」誕生以來，過去 17 年來的每一筆交易與附隨資訊，都被永久記錄在分佈於全球數萬個獨立礦工節點組成的分散式帳本中。這世界上沒有任何個人、企業、政府，甚至是比特幣創辦人中本聰本人，有能力去刪除或修改任何一筆已經確認的歷史區塊。

在技術底層，這是透過以下嚴謹的數學與物理共識機制來實現的：

  • 密碼學鏈結 (Cryptographic Linking & Hashing)：每一個新的區塊都包含了前一個區塊的 SHA-256 雙重雜湊值（Parent Hash）。這意味著所有歷史資料都以極高強度的加密演算法串聯在一起，若要修改任何一個歷史區塊，後續所有的區塊雜湊值皆會失效，必須全部重新演算。
  • 去中心化多節點共識 (Distributed Proof of Work)：比特幣由遍佈全球的數萬個獨立伺服器（礦工）進行驗證，任何單一節點若企圖修改本地資料，其帳本將與全球共識帳本不符，隨即會被共識機制永久拋棄。若要強行修改，必須掌握全網超過 51% 的龐大算力（51% Attack），在物理上這需要難以想像的百億美元級別計算資源，是完全不可能達成的。

雖然比特幣的原生網路為了交易安全，每一筆交易所能攜帶的資料極其微小（例如 OP_RETURN 腳本標記被限制在 80 位元組），但 SuperArchive 所採用的底層架構（Arweave）是基於完全相同的去中心化共識物理特性，並透過「區塊編織（Blockweave）」技術在數學上進行了優化，專門用於永久儲存大型檔案（如 PDF、圖片、壓縮檔）。檔案一旦寫入，即在全球多節點網路中硬編碼，任何人均無法將其更新、刪除或進行任何審查。

在本平台付費方案中，1 點 S-Point 售價為 5 NTD，可備份一塊 16MB 的檔案。若要儲存 1024MB (1GB) 的檔案，僅需 64 點 S-Points，換算下來是一次性付費 320 NTD，且未來永遠不需支付任何續期租金。

🦞 我的 AI 助理（如「養龍蝦 / Lobster」）該如何整合 SuperArchive API？

如果您希望為您的 AI 助理（例如 「養龍蝦 (Lobster)」）提供區塊鏈永久存檔能力，只需透過我們的開發者 API 即可無縫整合。您可以直接在系統中給予「龍蝦」以下指引步驟：

  • 步驟 1（取得金鑰）：請先登入本平台的用戶儀表板，在「API 金鑰管理」區域建立一個專屬金鑰（開頭為 sa_live_）。
  • 步驟 2（餘額檢查）：在指引龍蝦上傳任何重要檔案前，**必須先請龍蝦調用 `GET /api/v1/developer/points` 查詢當前剩餘點數**，確保您的 S-Points 餘額充足（每 16MB 檔案扣抵 1 點）。
  • 步驟 3（上傳封存）：引導龍蝦將需要永久備份的檔案，以 Form 表單 POST 發送至 POST /api/v1/SuperArchive 網址。

不可篡改之安全聲明：一旦龍蝦成功完成上傳，檔案將直接被廣播並簽署至全球區塊鏈。**由於去中心化的物理特性，上傳後連龍蝦（AI 助理）自己、甚至本平台管理員，都絕對無法刪除或修改該檔案。**


🤖 OpenClaw AI 雙技能工作流規範 (OpenClaw AI Multi-Skill Workflow)

為使您的 OpenClaw AI 助理 / 龍蝦 能百分之百自主控制流程、檢索剩餘點數、並執行上傳，您必須同時為其導入以下 兩個 核心 Tool / Skill 模組，並在其 System Prompt 中定義明確的「點數校驗與呼叫順序」：

  1. 優先呼叫 superarchive_check_points：讓 AI 連接 API，查詢當前授權 Key 綁定之 S-Points 餘額及使用狀態。
  2. 執行容量演算：AI 讀取本地待封存檔案位元組大小後，自主計算所需點數（檔案大小 / 16MB，不足 16MB 以 1 點計）。
  3. 餘額比對與例外中斷：比對剩餘 S-Points。若足夠則呼叫上傳技能；若點數不足，AI 必須立即中斷程序並向用戶警示餘額不足，指引至系統購買點數。
  4. 調用 superarchive_permanent_upload：在確認點數足夠安全無虞後，調用上傳技能。

🌐 為什麼本平台能保證「永久、永續」檔案儲存？

SuperArchive 閒貓典藏 底層封存機制完全基於 Arweave 去中心化區塊鏈協議。Arweave 協議採用數學上嚴謹設計之儲存捐贈基金（Storage Endowment）模型，當您上傳檔案時，所需的儲存費用已一次性預付至去中心化共識基金中，這確保了底層礦工在未來的數百年裡皆能獲得持續的利息收益來維持硬體運轉，進而保證檔案能安全保存超過 200 年（及理論上永久）。

全球頂尖的去中心化公鏈如 Solana，其全部的歷史帳本數據皆是唯一且排他性地儲存在 Arweave 區塊鏈上。這意味著，所有架構在 Solana 鏈上的數位資產，包括近期極具熱度的 Trump Coin (DJT 川普幣) 或是任何熱門代幣，其最終的永續交易歷史與鏈上憑證皆是藉由 Arweave 進行永久歸檔與背書的。若您希望深入了解 Arweave 兩百年的儲存安全保證與數學模型，可直接參閱官方公開的 Arweave 協議白皮書 (Arweave Yellow Paper)。

🔒 網頁端 AES-256-GCM 安全加密密碼是如何儲存的？

在「自行安全加密上傳（Option B）」模式下，本平台遵循嚴格的 **零知識 (Zero-Knowledge) 安全隱私規範**：我們 **絕對不會在任何伺服器資料庫或區塊鏈上記錄您的原始密碼**。

當您在網頁端輸入加密密碼時，我們會利用 SHA-256 演算法對您的密碼進行雜湊，衍生出高安全性的 256 位元 AES 對稱金鑰，並直接在您的瀏覽器記憶體中將檔案加密後才進行上傳。因為密碼僅存在於您的腦海中，**包含本平台管理員在內的任何第三方，都絕對無法解密、還原或讀取您的檔案。請務必妥善保存您的密碼，密碼一旦遺失，檔案將永久無法取回。**

🔓 為什麼有時候選擇「無加密上傳」是更好的？

無加密上傳非常適合那些您希望 **「永久對全球公開、透明且可被任何人隨時查閱」** 的檔案與數位資產。例如：

  • 您的個人榮譽、學術獎狀、畢業證書、結業證明.
  • 企業的產品認證、專利申請、營業登記、合約公開副本。
  • 開源軟體專案、學術論文、公開調查報告、歷史新聞紀錄。

選擇不加密時，檔案將以原始格式完全公開地存儲在區塊鏈上。任何瀏覽器、去中心化節點或區塊鏈瀏覽器（如 ViewBlock）皆能直接讀取、載入並渲染這些檔案，無須 any 密鑰授權，從而實現真正不可磨滅的永久公開存證。
