よくある質問 / FAQ

⛓️ なぜブロックチェーンに保存されたデータは「削除や変更が一切不可能」なのですか？

ブロックチェーンデータの改ざん耐性と永続性を理解するための最も完璧な例は、Bitcoin（ビットコイン）ブロックチェーンです。2009年にビットコインの「ジェネシスブロック（創世ブロック）」が生成されて以来、過去17年間にわたるすべての取引記録と付随メッセージは、世界中に分散された数万もの独立したマイナーノードによって永久に保存・検証され続けています。世界中のいかなる個人、企業、政府、あるいはビットコインの創設者であるサトシ・ナカモトであっても、承認済みの過去のブロックを削除したり書き換えたりする権限はありません。

ビットコインのネイティブネットワークは取引の安全性を最優先するため、1つの取引に含められるデータ容量は極めて微小（OP_RETURNスクリプトにより80バイトに制限）ですが、SuperArchiveが採用している基盤プロトコル（Arweave）は、全く同じ分散型合意形成の仕組みを用いながら、大容量ファイル（PDF、画像、圧縮アーカイブなど）の永久保存に特化して数学的に最適化された「Blockweave（ブロックウィーブ）」技術を搭載しています。一度書き込まれたファイルは、グローバルノードにハードコーディングされ、管理者や開発者を含む誰一人として更新、削除、あるいは検閲を行うことはできません。

📊 S-Pointsの料金体系（16MBで5 NTD）は、AWS S3やGoogle Driveと比較して高いですか、それとも安いですか？

当プラットフォームでは、1 S-Pointが5 NTDで販売されており、Arweave上で16MBブロックをアーカイブできます。1024MB (1GB)を保存する場合、必要なのは64 S-Pointsであり、これは一括で320 NTDの買い切りコストとなります。将来的な月額サブスクリプション費用は完全にゼロです。

この「一回限りの買い切り型」を、従来のクラウドストレージで1GBを1年、30年、50年保存した場合の維持費と比較すると、長期的な視点ではSuperArchiveが極めて安価であることが分かります。最も重要な違いは、従来のクラウドは支払いが遅れると、ファイルが警告なしに永久削除される点です。 比較表は以下の通りです：

料金モデル 一括支払い (買い切り) 毎回の継続請求 毎月/毎年の継続支払い
1年間の保存コスト (1GB) 320 NTD 約 9 NTD 約 6.5 NTD*

Google Driveは15GBの無料枠を提供していますが、利用規約には無料プランのデータについて消失等の責任を一切負わないと明記されており、不活性アカウント（2年間使用がない場合）はデータごとアカウントが自動削除されます。対してSuperArchiveは、数学的合意に基づいて、最初から削除不可能かつ改ざん不可能な終極の保管障壁をファイルに提供します。

🦞 AIエージェント（ロブスター / 養龍蝦など）をどのように連携させますか？

開発者APIを利用して、自動化されたAIアシスタント（例えば、ロブスター (Lobster) など）にブロックチェーンへの永久保存機能を簡単に統合させることができます。ロブスターに以下のプロセスを指示してください：

  • ステップ 1（APIキーの作成）：本プラットフォームのダッシュボードにログインし、「APIキー管理」セクションで専用のキー（接頭辞 sa_live_）を生成します。
  • ステップ 2（ポイント確認）：ロブスターがデータをアーカイブする前に、必ず GET /api/v1/developer/points を呼び出して、アカウントの「S-Points（ポイント）」残高が十分であることを確認させてください（16MBごとに1ポイントが消費されます）。
  • ステップ 3（永久アップロード）：ロブスターに、ファイルをマルチパート形式で POST /api/v1/SuperArchive へ送信させます。

不変性の警告：ロブスターがアップロードを完了すると、そのトランザクションは瞬時にブロックチェーンに永続的に刻まれます。**アップロードされたファイルは、ロブスター自身（AIアシスタント）や本プラットフォームの管理者であっても削除や変更を行うことは一切不可能です。**


🤖 OpenClaw AI 複数スキル連携ワークフロー (OpenClaw AI Multi-Skill Workflow)

AIエージェントが自主的に容量計算やポイントの事前照合を行うために、以下の2つのツールをエージェントに実装し、実行順序を指示してください：

  1. まず superarchive_check_points を呼び出す：APIを通じて、現在のアカウントに紐づくS-Points残高を取得します。
  2. 必要なポイント数を算出する：AIは対象ファイルの容量を読み取り、16MB単位で必要なスロット数を自主的に演算します（端数は切り上げ）。
  3. 残高不足時の処理：S-Points残高が必要ポイント数未満の場合、AIは処理を安全に中断し、ユーザーにポイントのチャージを促します。
  4. 点数適合後に superarchive_permanent_upload を実行：十分なポイントが確認できた場合にのみ、永久アップロードツールを呼び出します。

スキル 1：S-Points残高の確認

  "name": "superarchive_check_points",
  "description": "Query remaining S-Points balance and key status from SuperArchive using the Developer API Key.",
  "parameters": {
    "type": "object",
    "properties": {},
    "required": []
  }
}

スキル 2：永久アップロードの実行

  "name": "superarchive_permanent_upload",
  "description": "Upload a local file permanently to Arweave blockchain via SuperArchive. Auto-debits S-Points.",
  "parameters": {

🌐 なぜファイルの「永久」保存を保証できるのですか？

デジタル猫アーカイブ（SuperArchive）の基盤となる永久保存メカニズムは、分散型ブロックチェーンであるArweave（アーウィーブ）プロトコルに基づいています。Arweaveは、データストレージ基金（Storage Endowment）と呼ばれる数学的に構築された経済モデルを導入しており、ファイルをアップロードする際、200年以上（理論上は永久）にわたる物理的なストレージ費用が基金を通じて一括前払いされます。これにより、鉱工（マイナー）はデータを永続的に維持するインセンティブを得ることができます。

世界をリードする高速ブロックチェーンである Solana（ソラナ） も、取引履歴と元帳データをすべて永久アーカイブするためにArweaveを唯一のストレージとして採用しています。そのため、ソラナ上の著名なデジタル資産（Trump Coin (DJT)など）の永久履歴や証明書も、すべてArweave上で永続的に保護されています。200年間のストレージ安全保証と学術モデルの詳細については、公式の Arweaveプロトコル白皮書 (Arweave Yellow Paper) を直接ご参照ください。

🔒 Web UIの暗傷化パスワードはどのように保存されますか？

「自己管理型暗号化アップロード（Option B）」において、本プラットフォームは徹底した **ゼロ知識（Zero-Knowledge）プロトコル** を遵守しています。私たちは **お客様の生のパスワードをデータベースやブロックチェーンに一切保存しません。**

パスワードを入力すると、ブラウザ上で直ちにSHA-256を用いてハッシュ化され、強固な256ビットのAES対称鍵が導出されてファイルを暗号化したのち、初めてアップロードが実行されます。パスワードはお客様の記憶にのみ留まるため、**管理者を含むいかなる第三者もファイルを解号・復元することはできません。パスワードを紛失した場合、ファイルは永久に復元できなくなりますのでご注意ください。**

🔓 暗号化なし（No Encryption）を選択すべきなのはどのような場合ですか？

暗号化なしのアップロードは、**「世界中に永久に公開し、誰でもいつでも自由にアクセスさせたい」** ドキュメントやデジタル証明書に最適です。例えば以下のようなケースです：

  • 学術の表彰状、デジタル卒業証明書、保有スキルの認定証明。
  • 企業の製品安全性認証、特許の公開データ、契約の公開証明コピー。
  • オープンソースの成果物、歴史的ニュースのアーカイブ、公開調査レポート。

暗号化なしを選択した場合、ファイルは元のプレーンデータ形式で完全に公開ストレージに保存されます。キーの認証なしで、世界中のWebブラウザやブロックチェーンエクスプローラー（ViewBlockなど）から直接読み込み・表示・共有が可能です。
